KEY DECISIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Decision 1: Plugin Storage
  ✓ Recommend: File-based plugins in `plugins/` + metadata in Supabase
  - Alternative: Plugins as Supabase Functions (harder to develop)
  - Alternative: Cloud-hosted plugins (security risk)

Decision 2: Plugin Language
  ✓ Recommend: Python for executor, TypeScript for API/UI
  - Alternative: Single language (less flexible)
  - Alternative: Support all languages (maintenance burden)

Decision 3: Plugin Versioning
  ✓ Recommend: Git tags (v1.0.0, v1.1.0)
  - Alternative: Semantic versioning in plugin.json only
  - Alternative: No versioning (causes breakage)

Decision 4: Plugin Permissions
  ✓ Recommend: Declarative permissions + code review
  - Alternative: Full sandbox (too restrictive)
  - Alternative: No permissions model (security risk)
